Portfolio Optimization Approaches

Current Architecture

Heavy Initial Load

Everything is bundled into one big index.js. The browser downloads the code for the Projects page before showing the Intro.

Main Bundle (All Pages + Data)
  • ❌ Slow "Time to Interactive"
  • ❌ Image Pop-in (Layout Shift)
  • ❌ High CPU usage on startup

Recommended Architecture

Progressive Loading

Code is split into "Chunks". Heavy sections load in the background only when needed.

Intro (Loaded)
⬇️ On Scroll ⬇️
Intro
Projects (Loading Chunk...)
  • ✅ Instant initial paint
  • ✅ Smooth image transitions
  • ✅ Better mobile performance
